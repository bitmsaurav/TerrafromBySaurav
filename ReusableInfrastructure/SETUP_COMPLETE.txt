⚡ SUPER QUICK SETUP (8 steps)
================================================================================

1. EDIT CONFIGURATION
   Open: environments/dev/dev.tfvars
   Change these 6 values:
   - azure_subscription_id = "YOUR_ID"
   - project_name = "myproject"
   - owner = "yourname@company.com"
   - cost_center = "PROJ-001"

2. GENERATE SSH KEY
   mkdir -p ssh
   ssh-keygen -t rsa -b 4096 -f ssh/id_rsa -N ""

3. INITIALIZE TERRAFORM
   terraform init

4. PREVIEW DEPLOYMENT
   terraform plan -var-file=environments/dev/dev.tfvars | head -50

5. DEPLOY INFRASTRUCTURE
   terraform apply -var-file=environments/dev/dev.tfvars

6. GET OUTPUTS
   terraform output

7. TEST SSH ACCESS
   SSH_KEY=$PWD/ssh/id_rsa
   VM_IP=$(terraform output -json compute_outputs | jq -r '.vm_private_ips[0]')
   ssh -i $SSH_KEY azureuser@$VM_IP

8. CLEANUP (When done)
   terraform destroy -var-file=environments/dev/dev.tfvars

💡 IMPORTANT CONCEPTS
================================================================================

1. CONFIGURATION-DRIVEN DESIGN
   Don't modify .tf files - modify tfvars!
   All infrastructure defined in infrastructure_config
   
2. DYNAMIC NAMING CONVENTION
   <project>-<environment>-<region>-<resource>
   Example: myapp-dev-eastus-vm-1
   No naming conflicts across projects!

3. FEATURE TOGGLES (Enable/Disable Anything)
   enable_vm             = true   # Deploy VMs?
   enable_network        = true   # Deploy network?
   enable_key_vault      = false  # Deploy Key Vault?
   enable_monitoring = true   # Enable monitoring?
   All controlled from tfvars!

4. MODULE REUSABILITY
   Each module:
   • Works independently
   • Has clear inputs/outputs
   • Supports enable/disable
   • Has no hardcoded values
   = Truly reusable across projects!

5. ZERO HARDCODING
   No project names in code
   No subscription IDs
   No resource names
   No sensitive data
   → Everything configurable!
